# Contributing

Thanks for your interest in improving ShortStopper!

## Guidelines

- **Keep it stable** — Don't change behavior unless fixing a bug or security issue
- **Keep it focused** — Small, targeted changes are easier to review
- **Keep it minimal** — Avoid new permissions and external dependencies
- **Keep it safe** — Validate all data from storage or page contexts

## Before submitting

1. Test your changes by loading the extension unpacked in Chrome
2. Verify existing functionality still works
3. Run `node scripts/test-policy.mjs` if you modified route logic

## Pull requests

- Describe what the change does and why
- Reference any related issues
- Keep commits focused and well-described
