Why it works

Simple, predictable behavior: block the feed, keep the rest of the site usable.

Smart route policy

Centralized classification avoids overblocking and respects normal watch, results, and profile routes.

Overlay-first blocking

Overlay-first blocking prevents redirect loops and keeps SPA state stable.

Allow once + allowlist

Temporarily allow Shorts or whitelist channels without disabling the extension.

Security-first architecture

MV3, strict CSP, message validation, and local-only storage by default.

Security and privacy

ShortStopper uses least-privilege permissions and keeps data local.

Least privilege permissions

Hosts are scoped to supported sites. Optional adblock permissions are domain-limited.

No remote code

Everything ships with the extension. No external scripts or CDNs.

No analytics by default

ShortStopper does not track or collect browsing data.

FAQ

Quick answers to common questions.

Why is the Web Store recommended?

It provides automatic updates, verified distribution, and a smoother install experience.

What is blocked vs allowed?

Short-form feeds like Shorts and Reels are blocked. Search results, watch pages, channels, and profiles are allowed.

Does ShortStopper collect data?

No. Settings and stats live locally. Optional insights store domain-only counts locally.
